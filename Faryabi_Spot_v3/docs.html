smFISH spot detection · cell segmentation · interactive viewer

Overview

The Piscis pipeline is a complete end-to-end workflow for single-molecule fluorescence in-situ hybridisation (smFISH) data analysis. It combines two spot detectors, Cellpose cell segmentation, an interactive HTML viewer, and a web-based GUI — all designed to work together from raw TIF stacks to per-cell spot counts.

Installation

Conda environment (recommended)

# Create and activate
conda env create -f environment_piscis.yml
conda activate piscis_env

# Verify
python -c "import piscis, cellpose; print('OK')"

Manual install

pip install piscis
pip install "cellpose<4"          # pin to v3 for CP3-trained models
pip install flask tifffile scipy pillow numpy pandas scikit-image
⚠ Cellpose version
If your Cellpose model was trained with Cellpose 3 (most custom models are), you must install cellpose<4. Cellpose 4 changed the model format and is incompatible with CP3 weights.

GPU support

Piscis uses JAX. To use a GPU replace jax[cpu] with the appropriate CUDA backend:

pip install "jax[cuda12]"     # CUDA 12
pip install "jax[cuda11]"     # CUDA 11

SpotCaller uses only NumPy/SciPy and runs on CPU only. For large datasets it still benefits from --workers parallelization.

Input Files

Naming convention

Files must contain a channel keyword in their filename. The keyword is case-insensitive and can appear anywhere in the name:

KeywordChannelProcessing
Cy3RNA channel 1Spot detection
Cy5RNA channel 2Spot detection
DAPINucleiCell segmentation only

FOV grouping

Files are automatically grouped by position/FOV key. The channel keyword is stripped from the filename and the remainder becomes the FOV identifier:

Location_Cy3_xy001.tif   →  FOV: location_xy001  channel: Cy3
Location_Cy5_xy001.tif   →  FOV: location_xy001  channel: Cy5
Location_DAPI_xy001.tif  →  FOV: location_xy001  channel: DAPI

Supported formats

Multi-page TIF stacks (z-stacks). Tested with 16-bit grayscale single-channel stacks. Multi-channel TIFs are supported — the first channel is used.

Skipped files

Spot Detectors

🧠 Piscis

  • Deep-learning (JAX)
  • Fast on GPU
  • Requires trained model
  • Best for in-distribution data
  • Integer coordinates
  • Probability threshold (0–1)

🔬 SpotCaller

  • Classical LoG + Gaussian fit
  • No model needed
  • Adaptive per-tile noise
  • Sub-pixel localisation
  • Per-spot SNR, sigma, amplitude
  • MAD threshold (robust units)

SpotCaller algorithm

SpotCaller implements a four-stage detection pipeline:

  1. Background subtraction — Morphological opening with a disk kernel sized 3× your spot radius removes uneven illumination and autofluorescence gradients.
  2. LoG bandpass filter — Laplacian of Gaussian matched to the expected PSF (σ = spot_yx / √2) suppresses noise and amplifies blobs of the right size.
  3. Adaptive MAD threshold — The image is tiled into tile_size × tile_size blocks. In each tile the threshold is median + k × MAD. Tiles are bilinearly interpolated to a smooth full-image map. This means bright and dim regions get appropriate thresholds — a key advantage over global thresholding.
  4. Sub-pixel Gaussian fitting — Each candidate peak is fit with a 2-D asymmetric Gaussian + background using scipy.optimize.curve_fit. This gives sub-pixel (y_sub, x_sub) coordinates, fitted sigma_y/sigma_x, amplitude, and SNR. Spots that fail the quality filters (SNR, sigma bounds) are discarded.
✓ When to use SpotCaller
Use SpotCaller when you have no trained Piscis model, when imaging conditions differ substantially from the training data, or when you need sub-pixel localisation accuracy and per-spot quality metrics for downstream analysis (e.g. colocalization, intensity measurements).

Cell Segmentation (Cellpose)

When --cellpose_model is provided, the DAPI channel is used for cell segmentation. The pipeline always uses max intensity projection of the DAPI z-stack regardless of --projection, because max-projection gives the most complete nuclear signal for segmentation.

Spot-to-cell assignment

For each detected spot, the pipeline reads the Cellpose label mask at the spot's pixel coordinate. The cell ID at that coordinate is recorded. Spots that fall on background (label = 0) are counted as unassigned.

Output

Each FOV produces a *_cells.csv with one row per cell. Cells with 0 spots in every channel are still included — this is important for computing true negative rates.

⚠ Cellpose version
Custom models trained with Cellpose 3 (most lab-trained models) require cellpose<4. Install with pip install "cellpose<4".

Parallelization

The pipeline uses Python's ThreadPoolExecutor to process up to --workers FOVs concurrently (max 20). Each worker is an independent thread that handles one FOV from disk read through CSV write.

Thread safety

ComponentThread safetyNotes
SpotCaller✅ Fully thread-safePure NumPy/SciPy, no shared state
Piscis (JAX)⚠ Inference lockJAX is not fork-safe; a threading.Lock wraps model.predict() calls. I/O and preprocessing still parallelise.
Cellpose⚠ One instanceSingle Cellpose instance shared; each DAPI image processed sequentially in its FOV thread
CSV writes✅ No conflictsEach FOV writes to its own files. Summary CSV uses a threading.Lock

Outputs

  • <stem>_spots.csv One row per detected spot. Piscis: fov, channel, y, x. SpotCaller additionally: y_sub, x_sub, z_plane, snr, sigma_y, sigma_x, fit_residual, amplitude.
  • <stem>_cells.csv Per-cell spot counts. Columns: fov, cell_id, Cy3_spots, Cy5_spots. Cells with 0 spots are always included.
  • spot_counts.csv Per-FOV summary. Columns: fov, channel, n_spots, n_cells, tif_file.
  • cell_summary.csv Aggregate per-FOV and per-channel statistics: n_cells, total_spots.
  • run_YYYYMMDD_HHMMSS.log Full timestamped log of the run. Includes DEBUG messages, thread names, and full tracebacks on failure. One log file per run.
  • previews/<stem>.png 8-bit contrast-stretched image preview (--save_previews). Used by the HTML viewer.
  • previews/<stem>_mask.png Coloured cell boundary overlay PNG (--save_previews). Loaded automatically by the viewer.
  • <stem>_mask.npy Raw Cellpose integer label mask as a NumPy array for downstream use.

Tuning Guide

SpotCaller — step by step

  1. Measure spot_yx — open a preview in ImageJ/Fiji, draw a line across a bright spot, measure the FWHM in pixels. Divide by 2.35 to get sigma. Set --spot_yx to roughly this sigma value (typically 2–4 for smFISH).
  2. Set threshold — start at 5.0. Check the count in spot_counts.csv. If you're missing real spots, lower it. If you have debris and autofluorescence detections, raise it. The range 3–8 covers most datasets.
  3. Adjust min_snr — open the viewer, load a spots CSV, zoom into a region with suspect detections. Low-SNR spots (<2) are typically noise. Raise min_snr to 3–4 to clean up.
  4. Check sigma — in the SpotCaller CSV, look at the sigma_y and sigma_x distribution. Spots with sigma > 2× spot_yx are usually aggregates or out-of-focus debris. Lower max_sigma_ratio to reject them.

Piscis — step by step

  1. Start at threshold 0.5 — this is the probability cutoff from the model's sigmoid output.
  2. Lower to detect more spots — try 0.3 or 0.2 if you're missing dim spots.
  3. Raise to reduce false positives — try 0.6–0.8 if you see background detections.

Python API

SpotCaller drop-in replacement for Piscis

from spot_caller import SpotCaller

# Same interface as Piscis.predict()
model  = SpotCaller(spot_yx=3.0, threshold=5.0, min_snr=2.0)
spots  = model.predict(image_2d)   # returns (N, 2) float32 [y, x]

# Full output with quality metrics
df = model.detect(image_2d)
# df columns: y, x, y_sub, x_sub, z_plane, snr, sigma_y, sigma_x, amplitude

# Post-filter by SNR
confident = df[df.snr > 3.0]

# 3-D detection on a z-stack
df_3d = model.detect_3d(stack_zyx)

FAQ

Why does the script skip my files?

Check the log file — skipped files are logged with SKIP. Common causes: the filename has no Cy3/Cy5/DAPI keyword, it starts with a dot (hidden file), or it has non-ASCII characters. Run ls -la in your input directory to check.

Piscis model not found — ValueError: Model X is not found

If using a named model like 20230905, Piscis downloads it from HuggingFace on first use. Check your internet connection. If using a local model, provide the full path.

Cellpose error: CP3 models not compatible with CP4

Your model was trained with Cellpose 3. Install pip install "cellpose<4".

SpotCaller detects too many spots

Increase --threshold (try 7–10), increase --min_snr (try 3.0), or decrease --max_sigma_ratio (try 1.8). Check if --spot_yx matches your actual spot size — if it's too large it will accept wide blobs.

SpotCaller misses dim spots

Lower --threshold (try 3.0), lower --min_snr (try 1.5). Also check that --spot_yx is not too small — underestimating spot size reduces the LoG response.

The viewer won't load my TIF files

Browsers cannot decode scientific multi-page 16-bit TIFs. Run the pipeline with --save_previews to generate 8-bit PNGs, then load those instead. The pipeline PNG previews use the same best-focus plane / projection as detection.
